# Support Impact Release Checklist

## Purpose

Use this document when a ticket appears shortly after a release, deployment, or configuration change.

## Review steps

1. Confirm whether the incident started after a known release window.
2. Check whether the product area and environment match the changed component.
3. Compare with incidents opened in the same period.
4. Prefer rollback, configuration validation, or deployment verification guidance when evidence points to release correlation.

## Useful support phrasing

When release correlation is strong, document that the next action should verify deployment state before pursuing unrelated user-level troubleshooting.
