# Internal Knowledge Base

This folder is the first knowledge source for the app's grounded AI recommendation flow.

The AI now uses two evidence sources:

1. Similar solved tickets from the ticket database
2. Internal guidance documents from this folder

## Folder purpose

- `SOPs`: standard operating procedures your team should follow
- `KnownIssues`: repeated incidents, known defects, recurring patterns
- `Troubleshooting`: practical verification and diagnostic guides
- `ReleaseNotes`: product or deployment changes that affect support decisions

## Authoring rules

- Keep one topic per file
- Use clear headings
- Keep error codes, browser names, environments, and external system names exactly as support teams use them
- Write short practical steps, not long policy text
- Prefer concrete verification steps and escalation rules
