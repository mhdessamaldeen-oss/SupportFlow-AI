# Ticket Triage Playbook

## Initial validation

Confirm the reported issue, affected product area, environment, and user impact before proposing a fix. When reproduction is unclear, ask for one concise verification step rather than several low-value questions.

## Cross-language support handling

For Arabic and English tickets, keep the technical recommendation in one consistent support language internally and preserve the user-facing language in final responses. Error codes, browser names, environment labels, and external system names should be kept exactly as provided.

## Escalation signal

Escalate when the issue affects multiple users, blocks production workflow, or repeats after a prior resolution. A repeated ticket with the same product area and environment should be compared with previously resolved incidents before assigning new engineering work.
