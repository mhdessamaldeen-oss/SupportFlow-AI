# Ticket Triage SOP

## Purpose

Provide a consistent first-pass workflow for new support tickets before classification, escalation, or resolution recommendation.

## Intake checks

1. Confirm the ticket title and description clearly state the reported problem.
2. Confirm the affected product area, environment, browser, operating system, and external system when available.
3. Confirm whether the issue affects one user, multiple users, or a business-critical workflow.
4. Confirm whether the ticket appears to repeat a previously resolved incident.

## First triage decision

- If the ticket is missing essential reproduction details, request one precise clarification instead of several broad questions.
- If the issue blocks production workflow or affects multiple users, treat it as elevated operational risk.
- If the issue mentions deployment, integration, access sync, or repeated failure after a previous fix, compare it with resolved tickets before assigning a new path.

## Arabic and English handling

- Preserve the original user language in the support process.
- Keep technical identifiers exactly as reported.
- Treat Arabic, English, and mixed-language tickets as equivalent for retrieval and evidence comparison.

## Escalation rule

Escalate when impact is broad, the issue repeats after a prior resolution, or the ticket indicates a likely service-side failure rather than a one-user problem.
