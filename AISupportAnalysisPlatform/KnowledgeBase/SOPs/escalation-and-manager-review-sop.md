# Escalation And Manager Review SOP

## Trigger conditions

Escalate for manager review when any of the following apply:

- multiple affected users
- production workflow blocked
- repeated incident after a prior fix
- strong customer frustration or explicit escalation request
- SLA risk or SLA breach

## Support action before escalation

1. Confirm impact scope and affected user count.
2. Check whether a similar resolved ticket already exists.
3. Check whether a known issue or recent release change matches the problem.
4. Record one clear technical assessment before handoff.

## Evidence required in escalation

- short issue summary
- current status and priority
- affected product area and environment
- strongest similar ticket reference if available
- strongest knowledge-base reference if available
- current next-step recommendation

## Avoid weak escalation

Do not escalate with only generic wording such as "system issue" or "needs investigation". Escalation should include at least one evidence-backed reason.
